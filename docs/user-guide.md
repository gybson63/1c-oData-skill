# Руководство пользователя

## Что такое 1c-oData-skill

Это **Telegram-бот**, который помогает получать данные из базы 1С:Предприятие без программирования и без обращения к специалистам. Вы просто пишете вопрос на русском языке — и бот отвечает.

Бот подключается к вашей базе 1С (бухгалтерия, зарплата, склад и т.д.) и может:
- Показывать списки сотрудников, контрагентов, организаций, товаров
- Находить документы по номеру, дате или содержимому
- Считать количество записей («Сколько сотрудников в компании?»)
- Показывать данные с фильтрами («Покажи документы за последний месяц»)

---

## Что нужно для начала работы

Вам **не нужно ничего устанавливать** на свой компьютер. Бот работает внутри Telegram.

**Попросите вашего IT-специалиста:**
1. Настроить и запустить бот (он должен быть уже развёрнут на сервере)
2. Дать вам **ссылку на бота** или его **имя пользователя** в Telegram (например, `@MyCompany1CBot`)
3. Убедиться, что нужные объекты 1С опубликованы для доступа через OData

После этого просто откройте бота в Telegram и нажмите **«Запустить»** (или отправьте команду `/start`).

## Как пользоваться ботом

### Первый запуск

Отправьте команду `/start`. Бот поприветствует вас и покажет список подключённых агентов (модулей для работы с данными).

```
🤖 Бот для работы с 1С

Подключённые агенты: odata

Просто напишите запрос, и я постараюсь помочь.

/refresh — обновить метаданные 1С
/status — статус агентов
/tokens — расход токенов текущей сессии
/clear — очистить историю диалога
/history — статистика истории
```

### Как задавать вопросы

Просто напишите вопрос обычным русским языком. Чем конкретнее вопрос — тем точнее ответ.

#### Справочники

| Ваш вопрос | Что покажет бот |
|---|---|
| Покажи список организаций | Все организации из базы |
| Какие у нас есть подразделения? | Список подразделений |
| Найди сотрудника Иванова | Результаты поиска по фамилии |
| Покажи все должности | Полный список должностей |

#### Документы

| Ваш вопрос | Что покажет бот |
|---|---|
| Последние документы реализации | Документы «Реализация товаров и услуг» |
| Покажи начисления зарплаты за март | Документы «Начисление зарплаты» за период |
| Документ УТ-001234 | Конкретный документ по номеру |

#### Подсчёт записей

| Ваш вопрос | Что покажет бот |
|---|---|
| Сколько сотрудников в базе? | Число — общее количество |
| Сколько документов реализации за май? | Число с учётом фильтра по дате |
| Количество контрагентов? | Число контрагентов |

#### Фильтрация и уточнения

| Ваш вопрос | Что покажет бот |
|---|---|
| Покажи сотрудников отдела продаж | Сотрудники с фильтром по подразделению |
| Документы за последнюю неделю | С фильтром по дате |
| Не удалённые контрагенты | Без помеченных на удаление |

> 💡 **Совет:** Если бот показал не то, что нужно — уточните вопрос. Бот помнит контекст диалога, поэтому можно писать уточнения коротко: «а за апрель?», «только первые 5».

### Команды бота

| Команда | Описание |
|---------|----------|
| `/start` | Приветствие и список доступных команд |
| `/status` | Статус подключения к 1С (работает / не работает) |
| `/refresh` | Обновить информацию о структуре базы 1С |
| `/tokens` | Показать расход токенов и стоимость текущей сессии |
| `/clear` | Очистить историю диалога и сбросить счётчик токенов |
| `/history` | Показать статистику по истории сообщений |
| `/metrics` | Подробные технические метрики работы бота |

## Понимание ответов бота

### Как выглядят ответы

Бот форматирует ответы в удобном виде. Вот примеры:

**Список справочника:**

```
Организации (3):

• ООО «Первый БИТ»
• ИП Иванов Иван Иванович
• АО «Рога и Копыта»

Всего: 3
```

**Список документов:**

```
Реализация товаров и услуг (3):

• №УТ-000123 от 15.03.2025 — ООО «Покупатель» — 150 000 р.
• №УТ-000122 от 14.03.2025 — ИП Сидоров — 43 200 р.
• №УТ-000121 от 13.03.2025 — АО «Ромашка» — 12 500 р.

Всего: 3
```

**Подсчёт:**

```
Всего сотрудников: 1 234
```

### Пагинация (длинные списки)

Если данных много, бот показывает первую часть списка и добавляет кнопку:

```
➡️ Следующие
```

Нажмите на неё — бот загрузит следующую порцию записей.

### Подпись с токенами

Под каждым ответом бот показывает компактную строку:

```
📊 📥3,200 📤1,500 | 💰₽2.15
```

Это информация о потреблении ресурсов AI за текущую сессию:
- 📥 — входящие токены (размер запроса)
- 📤 — исходящие токены (размер ответа)
- 💰 — примерная стоимость

## Контроль расходов

### Что такое токены

Бот использует искусственный интеллект (AI) для понимания ваших вопросов и форматирования ответов. Работа AI оплачивается по количеству **токенов** — это условные единицы измерения текста.

Чем длиннее вопрос и ответ — тем больше токенов расходуется. Обычно один простой запрос стоит доли копейки.

### Команда /tokens

Отправьте `/tokens`, чтобы увидеть подробный отчёт за текущую сессию:

```
📊 Токены текущей сессии

  • Запросов: 5
  • Входящих (input): 3,200
  • Исходящих (output): 1,500
  • Всего: 4,700
  • Стоимость: $0.0450 (₽2.15)
```

### Как сэкономить

- Формулируйте вопросы **конкретно** — бот потратит меньше токенов на уточнения
- Используйте `/clear` для сброса истории — длинная история увеличивает расход токенов
- Не запрашивайте «все данные» — лучше спросить про конкретный период или объект

## Советы для лучших результатов

### ✅ Хорошие вопросы

- «Покажи последние 10 документов реализации»
- «Сколько сотрудников в подразделении 'Отдел продаж'?»
- «Найди контрагента ООО Ромашка»
- «Документы начисления зарплаты за январь 2025»

### ❌ Плохие вопросы

- «Дай данные» — слишком общо, бот не поймёт, какие именно данные нужны
- «Покажи всё» — слишком широкий запрос
- «Что нового?» — бот не отслеживает изменения, он показывает текущее состояние базы

### Используйте привычные термины

Бот понимает как технические названия из 1С, так и обычные слова:

| Вы говорите | Бот понимает |
|---|---|
| сотрудники | Справочник «Сотрудники» |
| контрагенты | Справочник «Контрагенты» |
| реализация, продажи | Документ «Реализация товаров и услуг» |
| зарплата, начисления | Документ «Начисление зарплаты» |
| отпуск | Документ «Отпуск» |
| больничный | Документ «Больничный лист» |
| приём на работу | Документ «Приём на работу» |

> 💡 **Для IT-специалиста:** Соответствие бизнес-терминов и объектов 1С настраивается в файле `bot/config_hint.md`. Если бот не понимает ваши термины — попросите IT-специалиста дополнить этот файл.

### Бот помнит контекст

В рамках одной сессии (до команды `/clear` или `/start`) бот помнит предыдущие сообщения. Поэтому можно вести диалог:

```
Вы: Покажи список сотрудников
Бот: (показывает список)

Вы: а сколько их всего?
Бот: Всего: 47

Вы: покажи только отдел продаж
Бот: (показывает отфильтрованный список)
```

## Частые проблемы и решения

### Бот не отвечает

**Возможные причины:**
- Нет подключения к интернету — проверьте сеть
- Сервер бота временно недоступен — подождите несколько минут и повторите
- Бот перезагружается — обычно занимает 10–30 секунд

**Что делать:** Подождите 1–2 минуты и отправьте вопрос повторно.

### «⚠️ Ошибка OData»

Это означает, что бот не смог получить данные из базы 1С.

**Возможные причины:**
- База 1С недоступна (перезагружается, нет связи с сервером)
- Нужный объект не опубликован для доступа через OData
- Временная сетевая проблема

**Что делать:**
1. Попробуйте повторить запрос через пару минут
2. Если ошибка повторяется — сообщите IT-специалисту
3. Попробуйте команду `/refresh` для обновления информации о базе

### «⚠️ Ошибка AI»

Искусственный интеллект временно недоступен.

**Что делать:** Подождите немного и повторите запрос. Если ошибка повторяется — сообщите IT-специалисту.

### Ответ обрезан («сообщение сокращено»)

Telegram ограничивает длину одного сообщения. Если данных слишком много, ответ обрезается.

**Что делать:**
- Уточните запрос: добавьте фильтр, период или ограничение
- Запросите меньший объём: «покажи последние 5» вместо «покажи все»

### Бот показывает не те данные

- Уточните вопрос — возможно, бот понял вас буквально
- Используйте более точные термины
- Попробуйте переформулировать вопрос

### Данные устарели

Если в базе 1С были изменения, но бот их не видит:

- Отправьте команду `/refresh` — бот обновит информацию о структуре базы
- Данные, которые вы запрашиваете, всегда берутся из 1С в реальном времени (кроме метаданных)

## Глоссарий

| Термин | Простыми словами |
|--------|------------------|
| **OData** | Способ получения данных из 1С через интернет. Бот использует его для запросов к базе |
| **AI (ИИ)** | Искусственный интеллект — бот использует его для понимания ваших вопросов на русском языке |
| **Токены** | Единицы измерения текста для AI. Влияет на стоимость работы бота |
| **Метаданные** | Информация о структуре базы: какие есть справочники, документы, их поля и реквизиты |
| **Сущность** | Объект в базе 1С — справочник, документ, регистр и т.д. |
| **Справочник** | Список однотипных объектов: сотрудники, контрагенты, товары, организации |
| **Документ** | Запись о событии: реализация, начисление зарплаты, приказ на отпуск |
| **Регистр сведений** | Таблица данных: курсы валют, штаты, планы |
| **Регистр накопления** | Таблица числовых показателей: остатки, обороты |
| **Агент** | Модуль бота, отвечающий за определённый тип задач (например, запросы к 1С через OData) |
| **Пагинация** | Разделение длинного списка на страницы с кнопкой «Следующие» |
| **Сессия** | Период работы с ботом от команды `/start` или `/clear` до следующей такой команды |
| **Профиль** | Набор настроек бота для конкретной базы 1С (настраивается IT-специалистом) |
| **Форматтер** | Модуль бота, который делает ответы красивыми и удобными для чтения в Telegram |

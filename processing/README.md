# Внешняя обработка: Включение интерфейса OData

## Назначение

Обработка позволяет быстро опубликовать объекты конфигурации 1С через стандартный
OData-интерфейс, не заходя в Конфигуратор.

## Структура файлов

```
EnableODataInterface/
  EnableODataInterface.xml          — метаданные обработки
  Forms/
    MainForm/
      Ext/
        Form.xml                    — структура формы
        Form/
          Module.bsl                — модуль формы (вся логика здесь)
```

## Как собрать EPF

Через скрипт (требуется 1С Конфигуратор и файловая или серверная база):

```powershell
powershell.exe -NoProfile -File epf-build.ps1 `
  -InfoBasePath "C:\Path\To\Your\Base" `
  -SourceFile ".\EnableODataInterface.xml" `
  -OutputFile ".\EnableODataInterface.epf"
```

Скрипт `epf-build.ps1` доступен в репозитории [cc-1c-skills](https://github.com/Nikolay-Shirokov/cc-1c-skills).

Или вручную через 1С Конфигуратор: Файл → Загрузить внешнюю обработку из XML → выбрать `EnableODataInterface.xml`.

## Использование

1. Откройте обработку в 1С: Файл → Открыть → `EnableODataInterface.epf`
2. Установите флажки для нужных типов объектов (по умолчанию — все)
3. Нажмите **«Включить OData»**
4. Дождитесь сообщения «Готово. Опубликовано объектов: N»

## Требования

- 1С:Предприятие 8.3.6+
- Права администратора в информационной базе
- База должна быть опубликована на веб-сервере с включённым OData

## Логика обновления состава

При открытии обработка вызывает `ПолучитьСоставСтандартногоИнтерфейсаOData()` и
проставляет флажки по типам — отмечены те, которые уже опубликованы.

При нажатии **«Включить OData»** обработка применяет следующие правила:

| Флажок | Поведение |
|--------|-----------|
| ☑ Включён | Все объекты этого типа будут опубликованы |
| ☐ Снят | Все объекты этого типа будут убраны из OData |
| Тип не показан в форме (напр. ПланыОбмена) | Сохраняется текущее состояние без изменений |

Это означает, что если в базе уже были вручную опубликованы, например, `ПланыОбмена`,
они останутся опубликованными независимо от действий в форме.

## Замечание о производительности

Для больших конфигураций (500+ объектов) первое обращение к OData после публикации
может быть медленным — сервер строит метаданные.
